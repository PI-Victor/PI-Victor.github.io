2020-12-13 :: PI-Victor
#grpc  #go  #protobuf  #reflection  #grpcurl  #debugging  #tips 
Earlier this year i started to write a lot of gRPC services and after a while fiddling around, there are things that are worth mentioning for the uninitiated. Design documentation I found the Google API design documentation useful for making you adopt some standards for your APIs early on. You don’t have to follow it word by word, but overall it definitely has its use. From mapping http methods to gRPC methods to naming conventions that help you design your API intuitive for future growth.
